Workspace setup for Fairspace

Your release is named {{ .Release.Name }} and installed into the namespace {{ .Release.Namespace }}.
It might take a few minutes for all pods to be ready. You can check it with the following command:

  kubectl --namespace={{ .Release.Namespace }} get pod

All pods should have status 'Running'

----------------------------

Your workspace is named {{ .Values.workspace.name }} and can be accessed via

{{- if .Values.workspace.ingress.enabled }}

  - http{{ if $.Values.workspace.ingress.tls.enabled }}s{{ end }}://jupyterhub.{{ .Values.workspace.ingress.domain }}

{{- else }}

* Run these commands in the same shell:

  export NODE_PORT=$(kubectl get --namespace {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services proxy-public)
  export NODE_IP=$(kubectl get nodes --namespace {{ .Release.Namespace }} -o jsonpath="{.items[0].status.addresses[0].address}")
  echo http://$NODE_IP:$NODE_PORT

{{- end }}

The installation contains an instance of JupyterHub.

----------------------------

A user has been setup in hyperspace {{ .Values.hyperspace.domain }} that is able to login to the workspace.
You can find the credentials for this user with the following commands:

kubectl get secret --namespace {{ .Release.Namespace }} testuser-secret -o jsonpath="{.data.username}" | base64 --decode; echo
kubectl get secret --namespace {{ .Release.Namespace }} testuser-secret -o jsonpath="{.data.password}" | base64 --decode; echo
